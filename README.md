# M-Flow × OmniCPG (Unified Code Memory & Property Graph Engine)

This project integrates **M-Flow** (a semantic memory knowledge graph layer) and **OmniCPG** (a static analysis Code Property Graph engine) into a unified codebase.

Both systems now share a single **Neo4j** graph database backend and are configured through a single unified `config.yaml` file.

---

## Repository Structure

- `docs/`: Merged documentation and synergy design concepts.
- `src/m_flow/`: M-Flow core codebase (semantic indexing, recall, memory pipeline).
- `src/omnicpg/`: OmniCPG core codebase (tree-sitter parsers, control flow, data flow).
- `src/mflow_cpg/`: Thin integration layer.
  - `config.py`: Unified configuration manager (`config.yaml`).
  - `chunker.py`: `SyntaxAwareCodeChunker` for method/class-based M-Flow ingestion.
  - `retriever.py`: `CPGRetriever` implementing `CODE_GRAPH` recall.
  - `linker.py`: `ConceptToCodeLinker` to connect semantic concepts with CPG nodes.
  - `semantic_engine.py`: Extensible hierarchical semantic enrichment engine.
  - `mcp_server.py`: Combined MCP server.
- `tests/`: Combined test suites.

---

## Configuration

All configuration is defined in `config.yaml` at the root of the project. Copy `config.yaml.example` to start:

```bash
cp config.yaml.example config.yaml
```

Under `semantic_analysis.levels`, you can configure and extend the hierarchical levels of semantic summaries generated by LLM (such as `high_level` for classes/modules and `details_level` for methods).

---

## Quick Start (with Ollama and Neo4j)

1. Run Neo4j locally or in Docker.
2. Run Ollama with your chosen models (e.g. `llama3` and `nomic-embed-text`).
3. Setup `config.yaml` with your connection parameters and model names.
4. Install dependencies:
   ```bash
   pip install -e .
   ```
5. Run tests:
   ```bash
   pytest tests/
   ```
